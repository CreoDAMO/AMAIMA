The error you're seeing when running `npm run lint` (which calls `next lint`) is caused by your legacy `.eslintrc.json` file. Next.js 14+ and ESLint 8+ (which you have via `eslint-config-next@^16.1.4`) use **flat config** (`eslint.config.js`) by default. The old `.eslintrc` format triggers internal deprecated flags like `useEslintrc`, `extensions`, `resolvePluginsRelativeTo`, etc., which ESLint v8+ has removed. `next lint` is strict and fails hard on these.

### Step-by-Step Fix
1. **Delete the Legacy Config**:
   - Remove `amaima/frontend/.eslintrc.json` (and any other `.eslintrc*` files like `.eslintrc.js` if present). Commit this change.

2. **Create Modern Flat Config**:
   - Add `amaima/frontend/eslint.config.js` (new file):
     ```js
     // eslint.config.js - Flat config for Next.js + TypeScript
     import nextPlugin from '@next/eslint-plugin-next';
     import tsEslint from '@typescript-eslint/eslint-plugin';
     import tsParser from '@typescript-eslint/parser';

     export default [
       {
         files: ['**/*.{js,jsx,ts,tsx}'],
         ignores: ['.next/**', 'node_modules/**'],  // Standard ignores
         plugins: {
           '@next/next': nextPlugin,
           '@typescript-eslint': tsEslint,
         },
         languageOptions: {
           parser: tsParser,
           parserOptions: {
             project: './tsconfig.json',  // Required for type-aware rules
             tsconfigRootDir: import.meta.dirname,
           },
         },
         rules: {
           ...nextPlugin.configs.recommended.rules,
           ...nextPlugin.configs['core-web-vitals'].rules,
           '@typescript-eslint/no-unused-vars': ['warn', { argsIgnorePattern: '^_' }],
           'no-console': 'warn',
           // Add any custom rules here
         },
       },
     ];
     ```

3. **Install Required Dependencies** (if missing):
   ```bash
   cd amaima/frontend
   npm install --save-dev @typescript-eslint/eslint-plugin @typescript-eslint/parser
   ```

   Your `package.json` already has `eslint`, `eslint-config-next`, and TypeScript supportâ€”good.

   ```bash
   npx eslint .
   ```

### Why This Fixes It
- Flat config avoids all deprecated flags (`useEslintrc`, `extensions`, etc.) that legacy `.eslintrc` injects.
- `next lint` uses your flat config automaticallyâ€”no need for `--dir` or other flags.
- TypeScript-aware rules (via `@typescript-eslint`) match your `.ts`/`.tsx` files.
- Keeps Next.js recommended rules (`core-web-vitals`) for best practices.
